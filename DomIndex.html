Preguntas

  1. ¿Qué es el DOM?

    2. El DOM (Document Object Model) es una estructura de árbol que representa el contenido de un documento HTML o XML en forma de objetos que se pueden manipular mediante JavaScript. Es una interfaz de programación de aplicaciones (API) que permite acceder y manipular los elementos de una página web.

  2. ¿Qué es un nodo?

    3. Un nodo es un objeto en el árbol DOM que representa un elemento, atributo o texto en un documento HTML o XML. Cada nodo tiene un tipo, un nombre y un conjunto de propiedades y métodos para acceder y manipular su contenido.

  3. ¿Cómo se accede a un elemento del DOM?

    4. Se puede acceder a un elemento del DOM utilizando los métodos getElementById, getElementsByClassName, getElementsByTagName o querySelector. Por ejemplo, para obtener un elemento con un id específico:

    const element = document.getElementById('mi-elemento');
  4. ¿Cómo se modifica un elemento del DOM?

    5. Para modificar un elemento del DOM se pueden usar los métodos innerHTML, setAttribute, classList, entre otros. Por ejemplo, para cambiar el contenido de un elemento:

    const element = document.getElementById('mi-elemento');
         element.innerHTML = 'Nuevo contenido';
  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. La diferencia principal es que getElementby* es una familia de métodos que seleccionan elementos del DOM por su nombre de etiqueta, id o clase, mientras que querySelector* es un método que selecciona elementos utilizando selectores CSS. Por ejemplo, para obtener todos los elementos <p>:

    const elementos = document.getElementsByTagName('p');

    Para obtener el primer elemento con una clase específica:

    const elemento = document.querySelector('.mi-clase');
  6. ¿Qué es un evento?

    7. Un evento es una acción que ocurre en un elemento del DOM, como hacer clic en un botón, mover el mouse sobre un elemento o cargar una página. JavaScript puede detectar estos eventos y responder a ellos ejecutr código específico cuando ocurren.

  7. ¿Qué es un listener?

    8. Un listener o "escucha" es una función que se registra para ser ejecutada cuando ocurre un evento en un elemento del DOM. Es decir, es una función que "escucha" un evento específico en un elemento y se ejecuta cuando ese evento ocurre. Para registrar un listener se utiliza el método addEventListener.

  8. ¿Qué es un callback?

    9. Un callback es una función que se pasa como argumento a otra función y que se ejecuta cuando la función original ha terminado de ejecutarse. En el contexto de los listeners, el callback es la función que se ejecuta cuando ocurre un evento. Por ejemplo, en el siguiente código el callback es la función anónima que se pasa como segundo argumento a addEventListener:

    const boton = document.getElementById('mi-boton');
            boton.addEventListener('click', function() {
            alert('Haz hecho clic en el botón!');
            });
  9. ¿Qué es un objeto de evento?

    10. Un objeto de evento es un objeto que se crea cuando ocurre un evento en un elemento del DOM y que contiene información relevante sobre ese evento, como el tipo de evento, el elemento que lo originó y los datos asociados con él. Este objeto se pasa como argumento al callback que se registra con addEventListener.
